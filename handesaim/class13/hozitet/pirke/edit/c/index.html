חשיבות הנושא

לאחר שבפרק הקודם "זרקנו את התלמידים למים" כדי שיטעמו עריכה ושימוש בתוכנת העריכה, בפרק זה נעשה סדר בהכרת הפונקציות של תוכנת העריכה, שאת חלקן למדו התלמידים תוך כדי התרגול בפרק הקודם. פונקציות אלה חשובות לצורך עריכה תיעודית ועלילתית.

תת-הנושאים ביחידה זו:

  • ג.1 שליטה בתוכנת העריכה מול שליטה במהות העריכה.
  • ג.2 הקשר שבין המחשב לתוכנת העריכה.
  • ג.3 הכרת תוכנת העריכה.

מטרות אופרטיביות

  1. התלמיד יבין ששליטה בתוכנת עריכה אינה שליטה בעריכה עצמה.
  2. התלמיד יבין מהי חשיבות הגיבוי.
  3. התלמיד יבין איך פועלת תוכנת העריכה בתוך המחשב.
  4. התלמיד יבין את הקשר שבין קובצי המדיה לבין הקליפים בתוכנת העריכה.
  5. התלמיד ידע לעבוד עם ממשק המשתמש של תוכנת העריכה.
  6. התלמיד ידע ליצור פרויקט ולאפיין אותו.
  7. התלמיד יכיר את תפקודי החלונות הראשיים.
  8. התלמיד ידע לעבוד עם ערוצי הקול.
  9. התלמיד יתרגל עם המורה את נושאי הפרק.

ג.1 שליטה בתוכנה מול מהות העריכה

הפעלת תוכנת העריכה והעריכה עצמה הם שני דברים שונים. תוכנת העריכה לעורך היא כמו תוכנת וורד לסופר. לא כל מי שיודע להפעיל את התוכנה יודע לערוך. בד בבד עם רכישת מיומנות טכנית, עלינו לרכוש מיומנויות של עריכה: בחירת טייקים, תיזמון (Timing), יצירת מתח, הרכבת דיאלוג ובניית סצנה.

ג.2 הקשר שבין המחשב לתוכנת העריכה

ג.2.1 Media Files ו-Source Clips

Media Files (קובצי מדיה): קובצי הווידיאו והאודיו שנמצאים בתיקייה במחשב. הם ה"מקור" הפיזי.

Source Clips: כאשר מייבאים מדיה לתוכנה, היא יוצרת "ייצוג" (קליפ) שמקושר לקובץ המקורי. הקליפ בתוכנה אינו הקובץ עצמו אלא רק מצביע עליו.

אם משנים את שם הקובץ המקורי או מזיזים אותו, התוכנה מאבדת את הקשר והקליפ הופך ל-Offline (מסך אדום). במקרה כזה יש לבצע Relinking (קישור מחדש).

ג.2.2 פרויקט הפרמייר

קובץ הפרויקט מכיל רק מידע: החלטות העריכה, סדר השוטים, אפקטים, וכדומה. הוא אינו מכיל את הוידאו עצמו. לכן, הוא שוקל מעט מאוד.

ג.2.3 חשיבות הגיבוי

חובה לגבות את קובצי המדיה ואת קובץ הפרויקט בכונן חיצוני. אל תסמכו על גיבוי אוטומטי במחשב בית הספר (שעלול להימחק).

ג.3 הכרת תוכנת העריכה

נכון ל-2022, הפרמייר כולל 4 מסכי עבודה ראשיים:

  • Project: אחסון חומרי הגלם והסיקוונסים.
  • Source Monitor: צפייה בחומרי הגלם (Source Clips).
  • Program Monitor: צפייה בתוצר הערוך (Sequence).
  • Timeline: שולחן העריכה, ייצוג גרפי של העריכה.
Bin: תיקייה בתוך חלון הפרויקט לארגון חומרים (כמו תיקייה במחשב, אבל בתוך התוכנה).
