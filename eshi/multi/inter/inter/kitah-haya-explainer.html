🎨
בונה שיעור
builder.html — כלי יצירת תוכן למורה
▾

הבונה הוא "סטודיו" פרטי של המורה. הוא יוצר שקפים, מגדיר שאלות הצבעה, משימות וטיימרים — ושומר הכל ל-Supabase. המורה יכול גם לגרור קובץ שיעור קיים ו-AI יחלץ ממנו שקפים אוטומטית.

למה הוא נפרד? כי בנייה היא עבודה יצירתית שלוקחת זמן — היא לא צריכה להיות על אותו המסך שממנו מנהלים שיעור חי.

ייבוא AI
גרירת קובץ HTML קיים → Gemini מחלץ שקפים אוטומטית
עורך חזותי
הוספת תמונות, יוטיוב, קופסות צבעוניות בלחיצה
שמירה מקומית
שיעורים נשמרים ב-localStorage ובנוסף ב-Supabase
הפעלה ישירה
כפתור "הפעל שיעור" פותח session חי מיד
זרימה: מורה בונה שיעור → לוחץ "הפעל" → Supabase יוצר session עם קוד 4 ספרות → המורה עובר ל-teacher.html עם הקוד

מסך מורה
teacher.html — שלט רחוק של השיעור
▾

זה "שלט הרחוק" של השיעור. המורה רואה את כל השקפים, מקדם בין שקפים, פותח הצבעות, מפעיל טיימרים — וכל פעולה נשמרת ל-Supabase ומגיעה לתלמידים תוך שניות.

למה הוא נפרד מהבונה? כי ניהול שיעור חי דורש ממשק פשוט ומהיר — לא את מכשירי העריכה. בחכינה תגיע ל-teacher.html רק עם הקוד הרלוונטי.

זמן אמת
כל שינוי מגיע לתלמידים תוך שניות דרך Supabase
תשובות חיות

4 מצבים
שידור / הצבעה / משימה / חופשי — בלחיצה אחת
משיכה
כפתור "משוך את כולם" מחזיר תלמידים חורגים לשקף
זרימה: מורה לוחץ "הבא" → JavaScript קורא ל-Supabase update → Supabase שולח event לכל המנויים → תלמידים רואים שקף חדש

מסך תלמיד
student.html — חלון לתוך השיעור
▾

התלמיד פותח את הדף, מכניס קוד 4 ספרות ושמו — וממתי ואילך הוא "מחובר" לשיעור. הדף מקשיב ל-Supabase ומתעדכן אוטומטית כשהמורה מחליף שקף.

למה הוא נפרד? כי התלמיד לא צריך לראות כלי ניהול. הממשק שלו פשוט ונקי — רק התוכן שהמורה בחר להציג ברגע זה.

כניסה קלה
קוד 4 ספרות + שם — אין צורך בחשבון
רענון אוטומטי
sessionStorage שומר פרטים — רענון לא מנתק
אינטראקטיביות
הצבעות, שליחת תשובות, טיימר נראה
מצב חופשי
כשהמורה פותח מצב חופשי — תלמיד עובד בקצב שלו
זרימה: polling כל 2 שניות + Supabase Realtime → כל שינוי ב-session מגיע ומעדכן את הדף אוטומטית
🗄️
Supabase
מסד הנתונים + עמוד השדרה של זמן האמת
▾

Supabase הוא הלב הפועם של המערכת. הוא שומר את ה-sessions (שיעורים פעילים), תשובות תלמידים והצבעות — ובנוסף מספק Realtime: כאשר המורה מעדכן שקף, Supabase מודיע לכל התלמידים המחוברים תוך אלפיות שניה.

Realtime
WebSocket שמעדכן את כל הלקוחות בו-זמנית
REST API
גישה פשוטה לנתונים בלי קוד שרת
חינמי בסיסי
המכסה החינמית מספיקה לשימוש רגיל של כיתה
טבלאות
sessions / students / votes / responses
טבלאות: sessions — session פעיל לכל שיעור | students — תלמידים מחוברים | votes — הצבעות | responses — תשובות לשאלות פתוחות
⚡
Cloudflare Worker
kitah-haya-gemini — גשר בין הדפדפן ל-AI
▾

דפדפן לא יכול לפנות ישירות ל-Gemini AI בגלל מגבלות אבטחה (CORS). ה-Worker הוא שרת זעיר שיושב בין הדפדפן ל-Gemini — מקבל את הבקשה, מעביר ל-Gemini, ומחזיר את התשובה לדפדפן עם הכותרות הנכונות.

למה Cloudflare? כי זה חינמי לגמרי עד 100,000 בקשות ביום, פרוס ב-200+ מיקומים בעולם, ורץ תוך אלפיות שנייה.

פותר CORS
מאפשר לדפדפן לפנות ל-API חיצוני בבטחה
חינמי
100K בקשות ביום בחינם — מספיק ל-250 מורים
ממיר פורמט
מקבל פורמט Anthropic, שולח פורמט Gemini
מהיר
פרוס קרוב לכל משתמש — latency נמוך
זרימה: builder שולח POST → Worker מקבל → Worker שולח ל-Gemini API → Gemini מחזיר JSON → Worker מחזיר לדפדפן
📦
GitHub Pages
אחסון ופרסום קבצי HTML בחינם
▾

כל קבצי HTML של כיתה חיה (builder, teacher, student) מאוחסנים ב-GitHub ומוגשים בחינם דרך GitHub Pages. כל עדכון ב-repository מתפרסם אוטומטית לאתר.

למה GitHub Pages ולא שרת? כי אין צורך בשרת — הלוגיקה כולה ב-JavaScript שרץ בדפדפן. GitHub נותן אחסון + CDN + HTTPS בלי עלות.

בונה שיעור

המורה בונה שיעור

לוחץ על סוגי שקפים, מזין תוכן, מוסיף שאלות הצבעה ומשימות. אפשר גם לגרור קובץ HTML קיים וה-AI יחלץ שקפים אוטומטית.

לוחץ "הפעל שיעור"

תלמידים מכניסים קוד

Supabase שולח event לכל הדפדפנים המחוברים → student.html מרנדר את השקף החדש אוטומטית.

תלמידים מגיבים

מצביעים, שולחים תשובות → נשמר ב-Supabase → המורה רואה תוצאות בזמן אמת.
